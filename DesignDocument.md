# Дизайн-документ

Для бота пользователи разделены на 3 типа: 
<ol>
 <li><i>Потребители</i> – пользователи, которые могут получать подсказки по задачам. Доступные функции: получение подсказок, отслеживание прогресса продвижения по задачам.</li>
 <li><i>Админы</i> – пользователь, имеющий всю полноту функций потребителя с дополнительной возможностью добавлять подсказки в базу. Доступные функции: получение подсказок, отслеживание прогресса продвижения по задачам, добавление и редактирование подсказок. Данное право пользователь может получить после накопления нужного рейтинга.</li>
 <li><i>Главный админ</i> – преподаватель. Может создавать, редактировать, удалять подсказки, следить за прогрессом других пользователей, снимать права администраторов</li>
</ol>

Общение с ботом начинается любым сообщением. После первого сообщения пользвателя, программа заносит запись о нём в базу данных. Дальнейшая работа бота заключается в редактировании полей прав, рейтинга и прогресса в таблице пользователей, а также в отправке пользователю необходимых данных из таблицы подсказок. У пользователя есть 2 типа запроса: первый-получение подсказки, второй - просмотр прогресса.

База данных содержит в себе 2 таблицы: таблица с данными о пользователях, таблица с данными о подсказках.

<h3>Данные о пользователях</h3>
<table>
 <tr>
  <td><b>Название поля</b></td>
  <td><b>Тип данных</b></td>
  <td><b>Описание</b></td>
 </tr>
 
 <tr>
  <td>ID пользователя</td>
  <td>INT</td>
  <td>Идентификационный номер пользователя в системе, в качестве значения берется ID пользователя на платформе ВКонтакте</td>
 </tr>

 <tr>
  <td>Прогресс по задачам</td>
  <td>CHAR</td>
  <td>Отсортированная форматированная строка, содержащая данные о решении пользователем задач (пример, '1-1;2-3;3-4;4-0;5-0')</td>
 </tr>

 <tr>
  <td>Тип пользователя</td>
  <td>INT</td>
  <td>0 - главный админ, 1 - админ, 2 - пользователь</td>
 </tr>

 <tr>
  <td>Рейтинг</td>
  <td>INT</td>
  <td>Рейтинг пользвателя в системе</td>
 </tr>
</table>

<h3>Данные о подсказках</h3>
<table>
 <tr>
  <td><b>Название поля</b></td>
  <td><b>Тип данных</b></td>
  <td><b>Описание</b></td>
 </tr>
 
 <tr>
  <td>ID подсказки</td>
  <td>INT</td>
  <td>Идентификационный номер подсказки в системе</td>
 </tr>
 
 <tr>
  <td>Задача</td>
  <td>INT</td>
  <td>Номер задачи, к которой относится подсказка</td>
 </tr>

 <tr>
  <td>Уровень</td>
  <td>INT</td>
  <td>Порядковый номер подсказки в очереди всех подсказок к данной задаче</td>
 </tr>

 <tr>
  <td>Вопрос</td>
  <td>CHAR</td>
  <td>Вопрос, верный ответ на который необходим для получения подсказки</td>
 </tr>

 <tr>
  <td>Ответ</td>
  <td>CHAR</td>
  <td>Верный ответ на вопрос для получения подсказки</td>
 </tr>

 <tr>
  <td>Ключевые слова</td>
  <td>CHAR</td>
  <td>Ключевые слова, разделённые точкой с запятой, в запросе пользователя, по которым выдаётся данная подсказка</td>
 </tr>

 <tr>
  <td>Условие выдачи</td>
  <td>CHAR</td>
  <td>qw - после ответа на вопрос, sq - по уровню, kw - по ключевому слову в вопросе</td>
 </tr>
</table>
